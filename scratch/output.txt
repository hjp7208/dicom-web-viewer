1. 한눈에 보기
이 시스템은 DICOM 의료영상을 업로드하면 AI 모델(ONNX)이 정상/이상을 판독하고, 그 결과를 새 DICOM 영상(SC)으로 만들어 돌려주는 Spring Boot 백엔드입니다. 프론트엔드는 업로드 화면, 검사 목록(대시보드), 그리고 원본·AI 결과를 띄우는 뷰어를 담당합니다.

2. 데이터 모델 (4계층)
DICOM 표준 그대로 4계층 구조입니다: Patient → Study → Series → DicomImage. 환자 한 명이 여러 검사를, 검사 하나가 여러 시리즈를, 시리즈 하나가 여러 영상을 가집니다.
계층
의미
프론트가 자주 쓰는 필드
Patient
환자 (비식별)
patientId, sex, birthDate
Study
검사 1건
studyInstanceUid, studyDate, studyDescription
Series
촬영 묶음
seriesInstanceUid, modality(CT/CR/MG…), bodyPart
DicomImage
영상 1장(슬라이스)
sopInstanceUid, instanceNumber, rows, columns, windowCenter/Width
중요 — 화면 표시에 꼭 필요한 값
rows / columns: 영상 크기. windowCenter / windowWidth: 밝기·대비. 이 값이 없으면 영상이 시커멓게 보일 수 있습니다.
s3Key(파일 위치)는 보안상 프론트에 직접 내려주지 않습니다. 영상은 항상 미리보기 API(PNG)로 받으세요.

3.1 업로드 — POST /dicomweb/upload
multipart/form-data로 .dcm 파일을 보냅니다. 폼 key는 files이며 여러 개 동시 전송 가능합니다. 응답은 저장된 영상 id 배열입니다.
요청 (FormData):
const fd = new FormData();
fd.append("files", file);   // 여러 장이면 반복 append
fetch("http://medical-env.eba-u78gwdph.ap-northeast-2.elasticbeanstalk.com/:5000/dicomweb/upload", { method:"POST", body: fd })
응답 200:
[1, 2, 3]   // DicomImage id 배열
3.2 파일 키 조회 — GET /api/ai/path/{id}
업로드로 받은 id를 추론 입력용 키(path)로 변환합니다.
응답 200:
{
  "id": 1,
  "sopUid": "2.25.123...",
  "path": "<study>/<series>/<sop>.dcm"   // 추론·미리보기에 그대로 사용
}
3.3 단일 추론 — POST /api/ai/infer
body의 dicomPath에 위에서 받은 path를 넣습니다. 추론 후 결과와 함께, 소견을 새겨 만든 SC 파일 키(scFile)를 돌려줍니다.
요청 (JSON):
{ "dicomPath": "<study>/<series>/<sop>.dcm" }
응답 200:
{
  "result": {
    "label": "정상(Normal)",        // 또는 "이상(Abnormal)"
    "confidence": 0.97,             // 0~1 (확신도)
    "probabilities": [0.97, 0.03]   // [정상, 이상]
  },
  "scFile": "ai-sc/2.25...dcm"      // 결과 SC 키 → 미리보기에 사용
}
주의
probabilities 순서는 [정상, 이상]입니다. 이상 확률은 probabilities[1].
confidence는 0~1 실수입니다. 퍼센트로 보이려면 ×100.
scFile은 AI 결과 영상의 키입니다. /api/ai/preview?path=<scFile>로 띄우세요.
3.4 일괄 추론 — POST /api/ai/infer/series/{seriesId} · /api/ai/infer/study/{studyId}
시리즈 또는 검사 전체 슬라이스를 추론하고 집계 결과를 반환합니다. 응답 형태는 두 API가 동일합니다.
응답 200:
{
  "seriesId": 10,         // study 추론 시에도 필드명은 seriesId (= 요청한 id)
  "total": 24,            // 슬라이스 수
  "abnormalCount": 2,     // 이상 의심 장수
  "maxAbnormal": 0.81,    // 최고 이상 확률
  "overall": "이상 의심",  // 또는 "정상"
  "slices": [
    { "instanceNumber": 1, "sopUid": "2.25...", "abnormal": 0.03, "label": "정상(Normal)" }
  ]
}
3.5 검사 목록 — GET /api/studies
대시보드·검사목록 화면용. 쿼리 파라미터는 모두 선택입니다.
파라미터
설명
keyword
검사설명·StudyInstanceUID·환자ID 부분일치 (예: Mammo)
modality
모달리티 정확일치 (예: CT, CR, MG)
응답 200 (배열):
[
  {
    "id": 1,
    "studyInstanceUid": "2.25...",
    "studyDate": "2026-06-20T10:30:00",
    "studyDescription": "Chest PA",
    "patientId": "ANON-0001",   // 비식별 값
    "modality": "CR",
    "imageCount": 1
  }
]
3.6 영상 메타데이터 — GET /image/{sopInstanceUid}
경로변수는 sopInstanceUid입니다. 단일 영상의 표시용 값을 반환합니다.
응답 200:
{
  "sopInstanceUid": "2.25...",
  "instanceNumber": 1,
  "rows": 2022, "columns": 2022,
  "windowCenter": 2048.0, "windowWidth": 4096.0
}
3.7 영상 미리보기(PNG) — GET /api/ai/preview?path={key}
DICOM을 PNG로 변환해 바이너리로 내려줍니다. 원본 키든 AI 결과(SC) 키든 동일하게 동작합니다. img 태그의 src에 URL을 그대로 넣으면 됩니다.
// 원본
<img src={`http://medical-env.eba-u78gwdph.ap-northeast-2.elasticbeanstalk.com/:5000/api/ai/preview?path=${encodeURIComponent(path)}`} />
// AI 결과(SC)
<img src={`http://medical-env.eba-u78gwdph.ap-northeast-2.elasticbeanstalk.com/:5000/api/ai/preview?path=${encodeURIComponent(scFile)}`} />
4. 표준 화면 흐름
4.1 AI 데모 흐름 (업로드 → 추론 → 표시)
백엔드에 이미 동작하는 흐름이며, 프론트 예제(app/ai-demo)도 이 순서로 구현돼 있습니다.
업로드: POST /dicomweb/upload → id 배열 수신 (id = ids[0])
키 조회: GET /api/ai/path/{id} → path 수신
추론: POST /api/ai/infer { dicomPath: path } → result + scFile 수신
표시: GET /api/ai/preview?path=scFile (또는 원본 path)로 영상 렌더링
4.2 목록·뷰어 흐름
검사 목록: GET /api/studies (keyword/modality 필터) → 대시보드 테이블
검사 선택 → POST /api/ai/infer/study/{studyId}로 일괄 판독 결과 표시
개별 영상 메타: GET /image/{sopInstanceUid}로 표시값 보강

6. 백엔드와 합의할 항목 (확인 필요)

확인 필요
로그인 API: 소셜 로그인(google/kakao/naver) 설정과 JWT 인프라는 있으나 /auth/** 컨트롤러가 소스에 없음 → 로그인 엔드포인트 스펙 확정 필요.
웹뷰어에 jwt 와 소셜 로그인이 필요한지?
2) 검사→영상 목록: 특정 Study의 영상 id/키 목록을 주는 조회 API가 없음. 현재 영상 키는 업로드 응답이나 추론 결과로만 확보 가능 → 뷰어용 목록 API 필요.

4) 페이지네이션: /api/studies가 전체를 한 번에 반환 → 데이터 많아지면 페이징 파라미터 필요.
5) study 일괄추론 응답의 첫 필드명이 seriesId임(값은 studyId). 혼동 주의 또는 백엔드에 명칭 정정 요청.
6.1 데이터 안전 원칙 (지켜야 함)
AI 결과는 원본을 덮어쓰지 않고 별도 SC(새 SOP UID)로 생성 — 현재 백엔드가 이 방식.
환자정보는 비식별 값(patientId)만 표시. 실데이터·미비식별 데이터 화면 노출 금지.
파일 위치(s3Key)·토큰을 URL이나 화면에 노출하지 않기.
